スキル & エージェント 機能ガイド

「スキル」は作業の型、「エージェント」は別働きの作業員。2つの役割と、ビジネス（ツール開発・資料作成）でのおすすめの使い方を一覧にまとめました。

まず全体像

スキル と エージェント の違い

この2つは役割が違います。混同すると「重いのに遅い」使い方になりがちなので、最初にここだけ押さえてください。

スキル（Skills）

= やり方の説明書

特定の作業手順・知識・書式をまとめたファイル。関連する作業が来るとClaudeが自動で読み込み、毎回同じ品質で出力します。

  • 出力を一定品質に保つ「型」
  • コンテキストは共有（同じ会話の中で実行）
  • 資料作成・定型業務・社内ルールの標準化に最適

エージェント（Subagents）

= 独立して動く作業員

独立したコンテキストで自律的にタスクを実行し、結果の要約だけを本体に返します。「ブラウザの別タブ」のイメージ。

  • 並列処理・大量ファイル探索が速い
  • 本筋の会話を汚さない／第三者視点でレビュー
  • トークン消費は大きい（後述）— 使いどころを選ぶ
機能一覧 ①

スキルの主な種類とおすすめ

「組み込み（最初から使える）」と「自作（型化する）」の2系統。資料作成は組み込みスキルだけでかなり進みます。

組み込み・資料作成

docx / pptx / xlsx / pdf

Word・PowerPoint・Excel・PDF の生成と読み取り。「この数字でパワポの提案資料を作って」で実ファイルが出力されます。

おすすめの使い方

提案資料・月次レポート・経営会議デッキの自動生成、既存ExcelやPDFからのデータ抽出・整形。

自作・型化

カスタムスキル（SKILL.md）

手順をまとめて .claude/skills/名前/SKILL.md に保存。name と description（200字以内）が必須で、descriptionの精度で自動起動が決まります。

おすすめの使い方

「月次レポート型」「料金プラン比較資料型」など、繰り返す作業を1度きれいに作って型化 → チームで同品質に量産。

自作支援

skill-creator

質問に答えていくだけでスキルの雛形を自動で組み立ててくれる、スキル作成用のスキル。

おすすめの使い方

最初の1個を作るとき。「いつもやっているこの作業を型にしたい」と相談するところから始める。

自動化

schedule（定期実行）

決まった時刻・周期でタスクを自動実行する仕組み。「毎朝」「毎週月曜」などの定型業務を無人化します。

おすすめの使い方

毎朝KPIを集計してSlackへ投稿、週次で競合動向を要約、月末に定型レポートを生成。

組み込み・共有

artifact-design / Artifacts

閲覧・共有できるWebページやダッシュボードを生成（このガイドもArtifactです）。

おすすめの使い方

比較表・集計ダッシュボード・社内向け説明資料を、URL1つでチーム共有。

組み込み・開発

code-review / verify / TDD ほか

差分レビュー、動作確認、テスト駆動開発、体系的デバッグなど、ツール開発の品質を支える開発系スキル群。

おすすめの使い方

社内ツールを作るとき、実装前に設計、実装後に /code-review で自己点検 → 品質を底上げ。

機能一覧 ②

エージェントの種類と使い方パターン

用途別の組み込みエージェントと、効果の高い4つの基本パターン。役割特化の自作エージェントも作れます。

組み込み・探索

Explore（読み取り専用探索）

大量のファイル・フォルダを横断して調べ、要約だけ返す。本体の文脈を汚さずに調査できます。

おすすめの使い方

「10ファイル以上を読む」調査。既存コードや大量ドキュメントの実態把握。

組み込み・設計

Plan（設計エージェント）

実装方針を立て、手順・重要ファイル・トレードオフを整理した計画を返します。

おすすめの使い方

新しい社内ツールを作る前の設計。いきなり実装せず「設計→実装→テスト」の入口に。

組み込み・汎用

general-purpose

調査・コード検索・複数ステップの作業をまとめて任せられる万能エージェント。

おすすめの使い方

「どこにあるか分からないものを探す」「独立した複数作業をまとめて進める」。

自作・役割特化

カスタムエージェント

.claude/agents/名前.md に定義。frontmatterで name description tools model などを指定し、使える道具を絞れます。

おすすめの使い方

「レビュー担当」「テスト作成担当」「セキュリティ点検担当」を用意し、本体は司令塔に徹する。

パターン

並列リサーチ / パイプライン

独立した調査を同時並行で。多段作業は「設計→実装→テスト」を段階ごとに別エージェントへ受け渡し。

おすすめの使い方

同時実行は3〜5体が目安。「独立作業が3つ以上」なら並列化を検討。

使わない場面：前段の全結果が必要な逐次作業、同じファイルの同時編集。
パターン

第三者レビュー

会話履歴の先入観なしに、独立した視点で「要件どおりか」「想定漏れがないか」を点検させる。

おすすめの使い方

納品・コミット前の最終チェック。資料なら「依頼内容を満たしているか」の検収役にも。

組み合わせレシピ

ビジネス用途別・おすすめの組み立て

スキル・エージェント・MCP連携を組み合わせると効果が跳ね上がります。御社の用途を想定した3つの型。

資料作成の量産

提案資料・月次レポート・会議デッキ

自作スキル（型）+ pptx / xlsx+ MCP（Drive）→ schedule で月末自動

社内ツール開発

業務効率化ツール・社内システム

Plan で設計→ TDD スキルで実装→ レビュー担当エージェント

定型業務の自動化

日次集計・問い合わせ対応・議事録

MCP（Slack / Gmail）+ 自作スキル+ schedule で毎朝実行
設定クイックリファレンス

どこに何を置くか

作り始めるときの置き場所と起動方法の早見表。チーム共有は社内リポジトリで配布します。

機能置き場所必須／主な設定起動方法
スキル .claude/skills/名前/SKILL.md name, description（200字以内） 関連作業で自動、または /名前
サブエージェント .claude/agents/名前.md name, description（+ tools / model） 自動委任、または明示指定
スラッシュコマンド .claude/commands/名前.md 手順を記述 /名前
プロジェクト文脈 CLAUDE.md 社内ルール・用語・対象システム 常時自動読み込み
スキルをエージェント化 SKILL.md の frontmatter context: fork を追加 スキル内容がそのまま別働きの指示に
2つの注意点。 ① エージェントの多用は単一セッションの約7倍のトークンを消費します。スキルやツール呼び出しで済む作業には使わず、並列・大量探索で効果が上回るときだけに。 ② 外部配布スキルにはプロンプトインジェクションのリスクがあるため、出所が確かなものだけを使用してください。
